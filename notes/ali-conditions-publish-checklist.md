# ALI conditions series — publish checklist (manual)

Use this after edits to journal, corpus notes, or FAQs so the live story stays aligned. No automation required unless you later choose to script overlap checks.

## Before you call a chapter “done”

1. **Grounding**: Open `notes/ali-conditions-corpus-map.md` and confirm the condition still matches its anchors.
2. **Overlap**: Read the other six chapters’ summaries—ensure this post does not steal another condition’s core argument.
3. **Front matter**: Title, slug, summary, `publish_date` (not future-dated for site inclusion), categories, takeaways as needed.
4. **Build the content index**: Run the project’s knowledge refresh step so the journal list and search index pick up the file (same routine as other journal updates).
5. **Spot-check**: Open the Journal area on the live site and read the rendered post once published.

## When scope expands

- **FAQ**: Add or edit sourced markdown under `ao-knowledge-hq-kit/faqs/`, then run the same knowledge refresh.
- **ALI marketing page**: Treat copy changes as their own small task—edit the page component, review on the live URL, ship with the same commit discipline as content.

## Optional later automation

Only if volume grows: consider a script that flags duplicate `slug` / same-week titles across journal—or reuse devotionals overlap rules for scripture if faith content appears in this series.
